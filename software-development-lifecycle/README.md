# Software Development Lifecycle

Central framework for GlowPowerRental software project delivery.

---

## Framework Alignment

This SDLC framework is aligned with three internationally recognized engineering standards:

| Standard | Scope | Role in Framework |
|----------|-------|-------------------|
| ISO/IEC/IEEE 12207 | Software Life Cycle Processes | Defines lifecycle phases and governance |
| ISO/IEC/IEEE 29148 | Requirements Engineering | Defines how requirements are elicited, analyzed, specified, and validated |
| ISO/IEC/IEEE 42010 | Architecture Description | Defines how system architecture is documented |

```
ISO/IEC/IEEE 12207
   |  lifecycle processes
   v
ISO/IEC/IEEE 29148
   |  requirements engineering
   v
ISO/IEC/IEEE 42010
   |  architecture description
   v
GlowPowerRental SDLC
```

---

## SDLC Phases

| Phase | Description | Current Documentation |
|-------|-------------|-----------------------|
| 1. Business Analysis | Identify business needs and opportunities | Covered |
| 2. Requirements Engineering | Elicit, analyze, specify, and validate requirements | **Fully documented** |
| 3. System Architecture | Define system structure, components, and interfaces | Planned |
| 4. Design | Detail component design and data models | Planned |
| 5. Development | Implement software components | Planned |
| 6. Testing | Verify and validate against requirements | Planned |
| 7. Deployment | Release to production environments | Planned |
| 8. Operations | Monitor, maintain, and support | Planned |

## Documentation Structure

| Looking for... | Go to | Example |
|----------------|-------|---------|
| A rule or convention | [standards/](standards/) | "What attributes must every requirement have?" |

| An explanation of how something works | [guides/](guides/) | "How does a business need become a software requirement?" |

| A copy-paste starting point | [templates/](templates/) | "Give me a BRD template" |
