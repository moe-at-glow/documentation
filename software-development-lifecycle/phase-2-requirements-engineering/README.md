# Phase 2: Requirements Engineering

Elicitation, analysis, specification, and validation of requirements for GlowPowerRental projects, aligned with ISO/IEC/IEEE 29148.

All documents follow the **QRH (Quick Reference Handbook)** format. For background/educational content, see the `training/` subdirectory.

---

## Phase Summary

| Attribute | Detail |
|-----------|--------|
| Entry Criteria | Approved Project Charter from Phase 1, sponsor and stakeholders identified, business need documented |
| Exit Criteria | Baselined BRD and SRS, populated RTM, all Must Have requirements validated, change management process established |
| Key Roles | Business Analyst, Product Owner, System Architect, QA Lead, Stakeholders |
| ISO Alignment | ISO/IEC/IEEE 29148 -- Requirements Engineering |

## Phase Entry Criteria

- [ ] Project Charter approved (Phase 1 exit criteria met)
- [ ] Sponsor and business owner identified
- [ ] Key stakeholders identified
- [ ] Business need documented and aligned with strategic objectives
- [ ] Phase 2 team and resources assigned

## Phase Exit Criteria

- [ ] Stakeholder analysis completed
- [ ] Business Requirements Document (BRD) baselined and approved
- [ ] Software Requirements Specification (SRS) baselined and approved
- [ ] Requirements Traceability Matrix (RTM) populated with BR-to-SWR links
- [ ] All Must Have requirements validated
- [ ] Change management process established
- [ ] Phase 3 team and resources identified
